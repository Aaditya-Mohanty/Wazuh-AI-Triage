# AI-Driven Threat Hunting with Wazuh and Llama 3

## 📖 Overview
This project automates the Tier 1 Security Operations Center (SOC) analyst triage process using Artificial Intelligence . It extracts security alerts generated by the open-source Wazuh SIEM , processes them, and securely queries Meta's **Llama 3** large language model (via Groq's high-speed API) to evaluate whether the alert is a potential security incident or a routine false positive .

The project also implements a modern **"Ruleset as Code" (RaC)** DevOps workflow, automatically pushing daily triage reports to GitHub to continuously document threat-hunting activities .

## ✨ Key Features
*   **Automated SIEM Alert Triage:** Automatically parses `alert.json` files generated by Wazuh .
*   **AI Analyst Integration:** Uses a custom SOC prompt to evaluate alerts, reducing human "alert fatigue" by filtering out benign anomalies .
*   **High-Speed Inference:** Connects to Groq's Language Processing Unit (LPU) API to run the `llama-3.3-70b-versatile` model at near-instant speeds .
*   **Ruleset as Code (RaC) Automation:** Utilizes Python's `subprocess` module to automatically stage, commit, and push daily security findings to this repository .

## 🔬 Academic Inspiration
This project was heavily inspired by recent academic research on using LLM agents for SOC Tier 1 triage. In rigorous testing, the Llama 3 architecture demonstrated the highest overall accuracy (91.4%) when classifying Wazuh alerts, proving its viability as a reliable SOC "co-pilot" .

*   A free [Groq API Key](https://console.groq.com/)
